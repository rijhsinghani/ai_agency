The workflow has TWO logical flows in a single JSON:

**Flow 1: Supabase pending_review → Slack Block Kit message**

Trigger: Supabase Trigger on `content_bank` table, filtering on INSERT or UPDATE where `status = 'pending_review'`

Action: Slack node sends a Block Kit message to #content-pipeline

**Flow 2: Slack button callback → respond immediately → update Supabase → trigger publishing**

Trigger: n8n Webhook node (receives Slack interactive callback POST)

CRITICAL ORDER (3-second timeout pattern):

1. Webhook node receives POST from Slack
2. "Respond to Webhook" node immediately sends `{"response_action": "clear"}` with HTTP 200
   — This MUST happen before any other processing
3. Parse button payload: extract `action_id` and `value` (content_bank_id)
4. Switch node on action_id:
   - `approve_all`: Update Supabase content_bank.status = 'approved', then execute sub-workflow trigger for ALL platforms
   - `approve_selected`: Update Supabase content_bank.status = 'approved' for selected platforms (handle via separate webhook with platform checklist)
   - `reject`: Update Supabase content_bank.status = 'rejected'
   - `edit`: No action needed (URL button opens Sheets directly)
5. Send follow-up Slack message confirming action (separate API call after responding)

**n8n node types to include:**

- `"n8n-nodes-base.supabaseTrigger"` — watches content_bank for status=pending_review
- `"n8n-nodes-base.slack"` — sends Block Kit message (resource: message, operation: post)
- `"n8n-nodes-base.webhook"` — receives Slack interactive callback
- `"n8n-nodes-base.respondToWebhook"` — sends immediate HTTP 200 (MUST be first node after webhook)
- `"n8n-nodes-base.set"` — parses the Slack callback payload (payload is URL-encoded JSON in body)
- `"n8n-nodes-base.switch"` — routes on action_id value
- `"n8n-nodes-base.supabase"` — updates content_bank status
- `"n8n-nodes-base.executionData"` — passes approved content_bank_id to publishing workflow (placeholder connection)

**Important implementation notes:**

1. Slack sends the interactive callback as `application/x-www-form-urlencoded` with a `payload` field containing JSON. The n8n Set node must decode it: `JSON.parse($body.payload)`

2. The "Edit in Sheets" button uses `url` instead of `value` — it opens the URL directly without a callback. No webhook handling needed for this action.

3. For "Approve Selected", the implementation in this plan is a simplified version that approves all platforms (same as "Approve All"). A future iteration can add a Slack modal for platform selection. Document this simplification in the workflow notes.

4. The Slack webhook URL must be registered in the Slack App's Interactivity & Shortcuts settings. The n8n webhook will have a URL like `https://n8n.your-vm.com/webhook/slack-content-approval`.

**Workflow JSON structure requirements:**

- Set `"name": "slack-approval-workflow"`
- Set `"active": false` (activated after credentials configured)
- Use credential placeholders: `"slackOAuth2Api": "SLACK_CRED_ID_PLACEHOLDER"`, `"supabaseCredentials": "SUPABASE_CRED_ID_PLACEHOLDER"`
- Include `"notes"` field with setup instructions:
  1. Import workflow into n8n

  3. Configure Supabase credential
  4. Copy the Webhook node URL, paste into Slack App → Interactivity & Shortcuts → Request URL
  5. Set SLACK_CHANNEL_ID in the Slack node parameters
  6. Activate workflow
  7. Test: manually update a content_bank row status to 'pending_review' in Supabase — should trigger Slack message within 60 seconds
- Include a `"notes"` attribute on the respondToWebhook node: "CRITICAL: This node MUST fire before any Supabase operations. Slack has a 3-second timeout on interactive callbacks."
