**Workflow design (two sub-workflows in one JSON):**

The workflow JSON must define two logical flows:

**Direction A: Supabase → Google Sheets (push on insert/update)**

- Trigger: n8n Supabase Trigger node watching the `content_bank` table for INSERT and UPDATE events
- Filter: Only process rows where `sync_source = 'supabase'` (skip sheets-originated updates)
- Action: n8n Google Sheets node — if `sheets_row_id` is null → Append Row; if `sheets_row_id` is set → Update Row at that row ID
- After write: Update the `sheets_row_id` in Supabase if a new row was appended (captures the Google Sheets row number)

**Direction B: Google Sheets → Supabase (poll every 5 minutes)**

- Trigger: n8n Google Sheets Trigger node (polling interval: 5 minutes), watching for row changes in the content_bank sheet
- Filter: Only process rows where `sync_source` column in the sheet equals `'sheets'` (user manually edited, not a system push)
- Action: n8n Supabase Update node — update the `content_bank` row matching the `id` column with changed fields
- After write: Set `sync_source = 'supabase'` to prevent bounce-back loop

**Anti-loop guard (critical):**
The `sync_source` field prevents infinite loops:

- When Supabase updates → Sheets, the Supabase row's `sync_source` stays `'supabase'`
- When Sheets updates → Supabase, the Supabase row's `sync_source` is set to `'supabase'` immediately after the write
- The Supabase trigger only fires on rows with `sync_source = 'supabase'` (the filter prevents re-triggering the Direction A flow for Direction B writes)

**Google Sheets column mapping:**
The sheet should have columns in this order (used by n8n node column mapping):
`id | brand | topic_title | hook | status | talking_points | platform_angles | visual_assets | service_package_links | sync_source | created_at | updated_at`

**n8n workflow JSON structure:**

Write a valid n8n workflow JSON with these node types:

- `"n8n-nodes-base.supabaseTrigger"` for Direction A trigger
- `"n8n-nodes-base.googleSheets"` (operation: append/update) for Direction A action
- `"n8n-nodes-base.googleSheetsTrigger"` for Direction B trigger
- `"n8n-nodes-base.supabase"` (operation: update) for Direction B action
- `"n8n-nodes-base.if"` nodes for sync_source filter checks
- Node connections defined in the `"connections"` field

Use placeholder credential IDs (e.g., `"supabaseCredentials": "SUPABASE_CRED_ID_PLACEHOLDER"` and `"googleSheetsOAuth2Api": "GOOGLE_SHEETS_CRED_ID_PLACEHOLDER"`) since credentials are configured per-environment in n8n settings.

Set the workflow name to `"content-bank-sync"` and `"active": false` (operator activates manually after credential setup).

Include a README comment block at the top of the JSON (in a `"notes"` field) explaining:

1. How to import: n8n Settings → Workflows → Import → paste JSON
2. Credentials to configure: Supabase API (URL + service role key), Google Sheets OAuth2
3. Google Sheets setup: Create a new Google Sheet named "Content Bank", share with the Google service account, paste the Sheet ID into the trigger node
4. How to activate: after credential setup, toggle workflow to Active
5. Anti-loop guard explanation

**Important:** The JSON must be valid JSON (no trailing commas, no comments — move explanatory text to `"notes"` field or node `"notes"` attributes). Validate mentally before writing.
