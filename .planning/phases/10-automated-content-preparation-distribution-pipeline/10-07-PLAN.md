**Step 1 — platform_analytics table:**

```sql
CREATE TABLE IF NOT EXISTS platform_analytics (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  content_bank_id UUID REFERENCES content_bank(id) ON DELETE CASCADE,
  platform TEXT NOT NULL CHECK (platform IN ('youtube', 'twitter', 'instagram', 'threads', 'substack')),
  post_id TEXT NOT NULL,           -- Platform-native post ID
  metric_date DATE NOT NULL DEFAULT CURRENT_DATE,

  -- Universal engagement metrics
  views INTEGER DEFAULT 0,
  likes INTEGER DEFAULT 0,
  comments INTEGER DEFAULT 0,
  shares INTEGER DEFAULT 0,
  saves INTEGER DEFAULT 0,

  -- Platform-specific metrics
  impressions INTEGER DEFAULT 0,   -- Twitter/YouTube
  retention_rate DECIMAL(5,2),     -- YouTube (0.00 to 100.00)
  ctr DECIMAL(5,2),                -- YouTube click-through rate
  reach INTEGER DEFAULT 0,         -- Instagram/Threads

  -- Metadata
  collected_at TIMESTAMPTZ DEFAULT NOW(),
  raw_response JSONB,              -- Full API response for debugging
  brand TEXT CHECK (brand IN ('sameer_automations', 'raj_photo_video')),

  UNIQUE(content_bank_id, platform, metric_date)
);
```

**Step 2 — ai_cost_tracking view:**

```sql
-- Create ai_usage table if it doesn't exist in this schema instance
-- (content-engine backend already uses this; ensure it exists here)
CREATE TABLE IF NOT EXISTS ai_usage (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  model TEXT NOT NULL,
  operation TEXT NOT NULL,  -- 'clip_analysis', 'content_generation', 'thumbnail_generation'
  input_tokens INTEGER DEFAULT 0,
  output_tokens INTEGER DEFAULT 0,
  estimated_cost_usd DECIMAL(10,6) DEFAULT 0,
  content_bank_id UUID REFERENCES content_bank(id) ON DELETE SET NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Monthly cost tracking view
CREATE OR REPLACE VIEW ai_cost_tracking AS
SELECT
  DATE_TRUNC('month', created_at) AS month,
  model,
  operation,
  COUNT(*) AS api_calls,
  SUM(input_tokens) AS total_input_tokens,
  SUM(output_tokens) AS total_output_tokens,
  SUM(estimated_cost_usd) AS total_cost_usd,
  ROUND(SUM(estimated_cost_usd)::numeric, 2) AS total_cost_rounded
FROM ai_usage
GROUP BY 1, 2, 3
ORDER BY 1 DESC, 5 DESC;
```

**Step 3 — Monthly cost summary view:**

```sql
CREATE OR REPLACE VIEW monthly_cost_summary AS
SELECT
  DATE_TRUNC('month', created_at) AS month,
  SUM(estimated_cost_usd) AS total_ai_cost_usd,
  COUNT(DISTINCT content_bank_id) AS videos_processed,
  CASE
    WHEN SUM(estimated_cost_usd) > 15 THEN 'OVER BUDGET'
    WHEN SUM(estimated_cost_usd) > 12 THEN 'WARNING'
    ELSE 'OK'
  END AS budget_status
FROM ai_usage
GROUP BY 1
ORDER BY 1 DESC;
```

Create `content-engine/n8n-workflows/analytics-nightly.json` — the nightly workflow that pulls engagement metrics from all platform APIs and stores them in Supabase.

**Workflow design:**

Trigger: n8n Cron node — runs daily at 2:00 AM (low traffic, platform APIs less rate-limited)

Flow:

1. Get all published content_bank rows
   - Supabase Select: `SELECT id, brand, visual_assets FROM content_bank WHERE status = 'published'`
   - This gives us the platform post IDs stored in visual_assets

2. For each published row, spawn parallel metric collection branches:

**Branch A: YouTube metrics**

- HTTP Request: `GET https://www.googleapis.com/youtube/v3/videos?part=statistics&id={youtube_video_id}&key={YOUTUBE_API_KEY}`
- Extract: viewCount, likeCount, commentCount, favoriteCount
- Insert/Upsert into platform_analytics: `(content_bank_id, 'youtube', youtube_video_id, metric_date=today, views, likes, comments)`

**Branch B: Twitter metrics**

- HTTP Request: `GET https://api.x.com/2/tweets/{tweet_id}?tweet.fields=public_metrics`
- Note: Free tier DOES NOT include read access. Tweet metrics require Basic tier ($100/month) or above.
- WORKAROUND: Log a warning "Twitter free tier does not support read access. Thread metrics not collected. Upgrade to Basic tier for analytics."
- Insert a placeholder row with 0 values and a note in raw_response: `{"error": "free_tier_no_read_access"}`

**Branch C: Instagram metrics**

- HTTP Request: `GET https://graph.facebook.com/v22.0/{instagram_post_id}/insights?metric=plays,likes,comments,reach&access_token={token}`
- Extract: plays → views, likes, comments, reach
- Upsert into platform_analytics

**Branch D: Threads metrics**

- HTTP Request: `GET https://graph.facebook.com/v22.0/{threads_post_id}?fields=id,text,views,likes,replies,reposts`
- Extract: views, likes, replies → comments, reposts → shares
- Upsert into platform_analytics

**Branch E: Substack**

- No public API for Note metrics → skip
- Log: "Substack Note metrics not available via API — check Substack dashboard manually"

3. After all branches complete: Update Google Sheets Analytics tab
   - n8n Google Sheets node: append new rows to the "Analytics" sheet in the content bank Google Sheet
   - Columns: Date | Topic | Brand | Platform | Views | Likes | Comments | Shares | Reach

4. Nightly cost check:
   - Supabase query the monthly_cost_summary view for current month
   - If budget_status = 'OVER BUDGET': send Slack alert to #content-pipeline
   - If budget_status = 'WARNING': log warning in n8n execution log
