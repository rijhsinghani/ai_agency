Create `content-engine/n8n-workflows/platform-publishers/youtube.json`.

**Workflow design:**

Trigger: Supabase Trigger on `content_bank` table for status change to 'approved'

Filter: Check that `visual_assets.processed_video` is not null (video is ready for upload)

**Long-form video upload flow:**

1. Generate GCS signed URL for the processed video (1-hour expiry)
   - Use n8n HTTP Request or Code node calling GCS signed URL API
   - URL: `https://storage.googleapis.com/...` with signed credentials

2. Upload to YouTube via YouTube Data API v3 using resumable upload
   - n8n HTTP Request node to `POST https://www.googleapis.com/upload/youtube/v3/videos?uploadType=resumable&part=snippet,status`
   - Body (snippet): `{title: platform_angles.youtube_long.title, description: platform_angles.youtube_long.description, categoryId: "27"}` (category 27 = Education)
   - Body (status): `{privacyStatus: "private"}` — upload as private, founder manually publishes after review
   - Header: `Authorization: Bearer {youtube_oauth_token}`
   - Log: "YouTube upload initiated. Quota cost: ~1,600 units (of 10,000 daily limit)"

3. After successful upload: update content_bank with YouTube video ID
   - Set `visual_assets.youtube_video_id = {videoId from API response}`
   - Update `status = 'published'` (or keep as 'approved' — log the published state)

**Shorts upload flow (sequential after long-form):**

4. Check if `visual_assets.clips` array is not empty
5. Take the highest `strength_score` clip from the array
6. Generate GCS signed URL for the clip file
7. Upload clip to YouTube via same videos.insert endpoint
   - Title: `{topic_title} #Shorts` (under 100 chars for Shorts)
   - Description: `{platform_angles.youtube_short}\n\n#Shorts\n\nBook a free 15-min discovery call: https://calendar.app.google/psycao3CrXjGnmk48`
   - Status: privacyStatus = "private" (same as long-form — manual publish after review)
   - The 9:16 aspect ratio + under 60s duration + #Shorts in description = auto-classified as Short by YouTube

**Important implementation notes:**

- YouTube OAuth2 requires user consent (cannot use service account for public channel upload). Use n8n's Google OAuth2 credential with YouTube Data API v3 scope (`https://www.googleapis.com/auth/youtube.upload`)
- Both uploads use `status.privacyStatus = "private"` so founder can review before making public. This is intentional — the system creates the video draft, founder reviews and publishes manually in YouTube Studio
- Quota logging is important: each upload costs 1,600 units; with 10,000 daily quota that's 6 uploads/day. Log this info in the workflow execution log
- Set `"active": false` on the workflow — activate manually after credentials are configured

**Workflow JSON structure:**

- `"name": "youtube-publisher"`
- Nodes: supabaseTrigger → if (has processed_video) → code (generate signed URL + format metadata) → httpRequest (YouTube upload long-form) → supabase (store video ID) → if (has clips) → code (get best clip) → httpRequest (YouTube upload Short) → supabase (store Short video ID)
- Credential placeholder: `"googleOAuth2Api": "YOUTUBE_OAUTH_CRED_ID_PLACEHOLDER"`
- Notes field: Setup instructions including OAuth2 scope requirement

Create `content-engine/n8n-workflows/platform-publishers/twitter.json`.

**Workflow design:**

Trigger: Supabase Trigger on `content_bank` table for status change to 'approved'

Filter: Check that `platform_angles.twitter` is not null and is a non-empty array

**Twitter thread posting flow:**

1. Extract tweet array from `platform_angles.twitter`
   - Expected: array of strings, each under 280 characters
   - If any tweet exceeds 280 chars, truncate to 277 + "..."

2. Post first tweet (standalone)
   - n8n HTTP Request to `POST https://api.x.com/2/tweets`
   - Body: `{"text": tweets[0]}`
   - Authentication: OAuth 1.0a (Twitter API Key + Access Token)
   - Store response `data.id` as `previous_tweet_id`

3. Loop over remaining tweets (tweets[1], tweets[2], ...)
   - n8n SplitInBatches node or Loop node over the remaining tweets array
   - For each tweet: POST /2/tweets with `{"text": tweet, "reply": {"in_reply_to_tweet_id": previous_tweet_id}}`
   - Store each response id as new `previous_tweet_id`
   - Add 1 second delay between tweets (rate limit protection)

4. After all tweets posted: update content_bank
   - Store first tweet's URL in `visual_assets.twitter_thread_url`
   - Log: "Twitter thread posted. Posts used: {tweet count} of 500 monthly allowance"

**Important implementation notes for n8n Twitter OAuth:**

n8n's built-in Twitter node uses OAuth 1.0a. The credential requires:

- Consumer Key (API Key)
- Consumer Secret (API Key Secret)
- Access Token
- Access Token Secret

All four values come from developer.twitter.com. The access token is generated for the account that will post (Sameer's @SameerAutomates account).

**Rate limiting awareness:**

- Free tier: 500 posts/month (hard limit)
- A 5-tweet thread = 5 posts. At 1 video/week with 5-tweet threads = 20 posts/month from threads
- Remaining ~480 posts/month for standalone tweets, replies, and other content
- The workflow should log post count per execution to track against the monthly limit
- No automated tracking of monthly usage needed in v1 — rely on developer.twitter.com dashboard

- `"name": "twitter-publisher"`
- Nodes: supabaseTrigger → if (has twitter array) → code (validate + format tweets) → httpRequest (post first tweet) → splitInBatches (remaining tweets) → httpRequest (post reply) → supabase (store thread URL)
- Credential placeholder: `"twitterOAuth1Api": "TWITTER_OAUTH_CRED_ID_PLACEHOLDER"`
- Notes: Rate limit warning (500 posts/month), OAuth 1.0a setup instructions
