  <action>
    Create `.claude/skills/youtube-thumbnail/SKILL.md` as a standalone skill focused exclusively on thumbnail generation. This skill is a focused subset of the full youtube pipeline skill.

    The SKILL.md must follow the Claude Code skill format:

    **Frontmatter:**
    ```yaml
    ---
    name: youtube-thumbnail
    description: Use when the user wants to generate 4 YouTube thumbnail variations for a video — uses Gemini 3 Pro Image Preview for generation, composes a 2x2 comparison grid, and presents options for selection. Accepts a video title and optional hook text.
    ---
    ```

    **Trigger line:**
    `Generate YouTube thumbnail variations: $ARGUMENTS`

    **Skill body (numbered phases):**

    Phase 1: Preflight
    - Check GEMINI_API_KEY is set (required)
    - Check HEADSHOT_DIR is set and contains at least one image
    - Accept input: video_title (required), thumbnail_hook (optional), content_bank_id (optional)
    - If invoked from a GCS trigger JSON (thumbnails-pending/ prefix), parse the JSON for video_title, brand, content_bank_id

    Phase 2: Design 4 Concepts
    - Work through desire-loop framework for this video title:
      1. Desire: What does the viewer want after watching?
      2. Pain: What frustration do they have?
      3. Transformation: What does the video deliver?
      4. Curiosity loop: What question does the thumbnail open?
    - Design 4 distinct concepts (A, B, C, D) varying: visual focus, text treatment, color direction, expression, composition
    - Present concepts to user before generating

    Phase 3: Generate 4 Thumbnails
    - Pick first headshot from $HEADSHOT_DIR
    - Run all 4 in parallel using the existing generate_thumbnail.py script:
      ```bash
      python3 .claude/skills/youtube/scripts/generate_thumbnail.py \
        --headshot "$HEADSHOT_DIR/<first_headshot>" \
        --thumbnail-hook "<concept hook text>" \
        --prompt "<concept-specific prompt>" \
        --output "youtube-thumbnails/<YYYY-MM-DD>/thumbnails/<video-slug>/a.png"
      ```
    - Repeat for b.png, c.png, d.png
    - Use the prompt template from the youtube skill (dark moody background, person on right 40% width, bold white text on left, 16:9 ratio)

    Phase 4: Compose 2x2 Grid
    - Use combine_thumbnails.py:
      ```bash
      python3 .claude/skills/youtube/scripts/combine_thumbnails.py \
        --images a.png b.png c.png d.png \
        --output comparison.png \
        --labels "A" "B" "C" "D"
      ```
    - Read and display the comparison grid to the user

    Phase 5: Select and Iterate
    - Ask user: "Which direction — A, B, C, or D? Or describe changes."
    - Regenerate if requested (pass --reference for variations)
    - On final selection, report the path to the selected thumbnail

    Phase 6: Update Content Bank (if content_bank_id provided)
    - If content_bank_id was provided (from GCS trigger), update the content_bank row:
      ```bash
      curl -s -X PATCH "$SUPABASE_URL/rest/v1/content_bank?id=eq.$CONTENT_BANK_ID" \
        -H "apikey: $SUPABASE_SERVICE_KEY" \
        -H "Authorization: Bearer $SUPABASE_SERVICE_KEY" \
        -H "Content-Type: application/json" \
        -d '{"visual_assets": {"thumbnail_grid_url": "<grid_url>", "selected_thumbnail": "<selected_path>"}}'
      ```

    **Key details:**
    - Scripts path: `.claude/skills/youtube/scripts/` (reuse existing scripts, do NOT duplicate)
    - Model: Gemini 3 Pro Image Preview (gemini-3-pro-image-preview)
    - Cost: $0 (Gemini free tier, 100 requests/day — 4 thumbnails = 4 requests)
    - Style: Dark moody backgrounds, purple/cyan accent palette, bold sans-serif headlines (per brand-style.md in youtube skill)

    **Troubleshooting table at end:**
    | Symptom | Likely cause | Fix |
    | No image generated | GEMINI_API_KEY missing | Set env var |
    | AI artifacts in faces | Prompt too complex | Simplify concept |
    | Script not found | Wrong working directory | Run from repo root |
