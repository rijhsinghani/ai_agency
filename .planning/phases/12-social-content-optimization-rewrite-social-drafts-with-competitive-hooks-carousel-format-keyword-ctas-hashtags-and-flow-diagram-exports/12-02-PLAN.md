  <action>
Create the carousel-generator tool at `research/tools/carousel-generator/`. This mirrors guide-generator's structure.

**1. package.json** — Same deps as guide-generator:

```json
{
  "name": "carousel-generator",
  "version": "1.0.0",
  "description": "Generate branded Instagram carousel slides and flow diagram social assets",
  "bin": { "sameer-carousels": "./index.js" },
  "dependencies": {
    "commander": "^14.0.3",
    "puppeteer": "^24.0.0"
  }
}
```

Run `cd research/tools/carousel-generator && npm install` after creating.

**2. lib/renderer.js** — Puppeteer PNG renderer:

- `renderPNG(htmlContent, outputPath, width, height)` function
- Use `page.setViewport({ width, height, deviceScaleFactor: 1 })` — exactly 1080x1080 output (not 2x retina, Instagram compresses anyway)
- `page.setContent(htmlContent, { waitUntil: "networkidle0" })` for Google Fonts loading
- `page.screenshot({ path, type: "png", clip: { x: 0, y: 0, width, height } })`
- `try/finally browser.close()` pattern (per Phase 11 decision)
- Share one browser instance per guide run (optimization): export `launchBrowser()` and `closeBrowser()` helpers, plus `renderPNGWithPage(page, html, path, w, h)` that reuses an existing page. The CLI orchestrates browser lifecycle.

**3. lib/slides.js** — Slide content registry for all 5 guides:

- Export `SLIDE_DATA` object keyed by slug
- Each guide has `keyword`, `title`, and `slides` array
- Each slide: `{ type, headline, subtext }` where type is one of: "hook", "problem", "math", "fix", "proof", "flow", "cta"
- Slide count varies per guide (5-8 slides per CONTEXT.md decision)
- Natural structure for most guides is 7 slides: hook, problem, math, fix, proof, flow diagram, CTA
- CTA slide headline: "DM me {KEYWORD}" with subtext including booking URL
- Use dollar-math pain anchors from guide data:
  - missed-call-textback: "$20,800/year" (5 calls/wk x $80 x 52wk)
  - review-automation: "25-30% more clicks" (8 to 94 reviews)
  - monday-pipeline: "$5,400/year" (3 leads/mo x $150 x 12mo)
  - quote-writer: "20 hours/month" (= 1 week billable)
  - no-show-killer: "$12,480/year" (3 no-shows/wk x $80 x 52wk)
- No emojis in any slide content

**4. lib/template.js** — `buildSlideHTML(slide, slideNum, totalSlides, guideSlug)`:

- Returns a complete HTML document string for one carousel slide
- Dark background (#1A1A1A), DM Sans font via Google Fonts CDN link
- Body: 1080x1080px, flex column, center content, 96px padding
- Slide number indicator: "01 / 07" at top in muted text
- Headline: #EDE9E3, 64px bold (72px on CTA slides), max-width 880px
- Dollar amounts and key numbers wrapped in `<span class="accent">` with purple-to-cyan gradient text
- Subtext: rgba(237,233,227,0.65), 28px, max-width 820px, margin-top 32px
- Brand bar: 4px gradient bar at absolute bottom
- Slide-type CSS variants via body class (e.g., body.slide-cta, body.slide-flow)
- For "flow" type slides: embed the SVG from getFlowDiagram(slug) with CSS scale transform to fit 1080px width. Import via `require(path.resolve(__dirname, "../../guide-generator/lib/template"))`. Wrap SVG in div with `transform: scale(1.3)` for square format.
- See RESEARCH.md "Pattern 2: Carousel Slide HTML Template" for the exact CSS structure

**5. lib/flow-template.js** — `buildFlowHTML(slug, format)`:

- Returns complete HTML for standalone flow diagram social asset
- `format` is "square" (1080x1080) or "landscape" (1200x675)
- Import `getFlowDiagram` from guide-generator: `require(path.resolve(__dirname, "../../guide-generator/lib/template"))`
- Dark background, centered SVG with CSS scale transform
- "How the automation works" label above the diagram in muted uppercase text
- Guide title below the diagram
- Brand bar at bottom
- Square: scale(1.3), Landscape: scale(1.5) for the SVG wrapper

**6. index.js** — CLI entry point:

- Commander-based: `--guide <slug>` for single guide, `--all` for all 5
- `--output <dir>` defaults to `./output/`
- For each guide: render all carousel slides + flow-square.png + flow-landscape.png
- Output structure: `output/{slug}/slide-01.png, slide-02.png, ..., flow-square.png, flow-landscape.png`
- Share one browser instance across all slides in a guide (open once, render all, close)
- Log progress: "Rendering {slug} slide {n}/{total}..."
- Exit with count summary: "Generated {n} slides + {m} flow diagrams for {k} guides"

**7. output/.gitkeep** — Empty file to preserve directory in git. Add `output/*.png` and `output/*/` to .gitignore (or create a local .gitignore in output/).

**Anti-patterns to avoid:**

- Do NOT use `os.tmpdir()` for paths (macOS issue from Phase 8-03)
- Do NOT use `headless: 'new'` — use `headless: true` (consistent with guide-generator)
- Do NOT use `deviceScaleFactor: 2` unless explicitly needed — 1x gives exact 1080x1080
- Do NOT duplicate SVG definitions — import from guide-generator
