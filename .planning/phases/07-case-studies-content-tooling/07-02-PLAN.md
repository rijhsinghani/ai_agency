```json
}
```

Run `npm install` in the CLI directory to install Commander.js.

**Step 3 — index.js (entry point):**

Line 1 must be `#!/usr/bin/env node`. Use `'use strict';`. Set up Commander with these flags:

- `-c, --count <n>` — number of topics (default: 5)
- `-f, --focus <vertical>` — filter by vertical/keyword
- `-m, --format <platform>` — output format: youtube, twitter, carousel (default: youtube)
- `--verbose` — show scores, penalties, de-dup reasons

Main action: resolveRoot() -> loadSources() -> parseTopicBank() -> markDuplicates() -> scoreTopics() -> formatOutput() -> stdout.write()

**Step 4 — lib/loader.js:**

- `resolveRoot()`: use `CONSULTANCY_ROOT` env var, fallback to `path.resolve(__dirname, '../../../..')` (4 levels up from lib/ to repo root)
- `loadSources(root)`: read topic-bank.md (required — throw on missing), content-plan.md (optional — warn and continue), ops/packages/\*.md (optional — warn and continue). All reads are synchronous `fs.readFileSync`.

**Step 5 — lib/parser.js:**

State machine parser for topic-bank.md. States: SCANNING, IN_TOPIC_SECTION.

For each `### ` heading (that is not in a ranked table): create new topic object. Extract fields:

- title from heading text
- baseScore from `**Score:**` line (parse float)
- subreddits from `**Subreddits:**` line (comma-separated)
- packageMatch from `**Package:**` line
- evidenceQuotes from lines starting with `> "`

After parsing, validate: `topics.length > 0` or throw descriptive error. Validate each topic has non-zero baseScore.

Read the actual `research/output/topic-bank.md` file structure to confirm the parser handles the real format. Adjust field extraction patterns if the actual file differs from the spec's expectations.

**Step 6 — lib/dedup.js:**

De-duplicate by core pain point category. Extract themes from content-plan.md using keyword matching:

```javascript
const COVERED_THEMES = [
  {
    theme: "bookkeeping",
    keywords: ["bookkeeping", "invoice", "accounting", "financial"],
  },
  {
    theme: "missed-call",
    keywords: ["missed call", "missed calls", "unanswered", "text-back"],
  },
  {
    theme: "employee-replacement",
    keywords: ["employee", "staff", "hire", "operations"],
  },
  {
    theme: "automation-framework",
    keywords: ["worth automating", "what to automate", "framework"],
  },
  {
    theme: "content-engine",
    keywords: ["content engine", "content pipeline", "social posts"],
  },
];
```

For each topic, check if its title or content angle matches any covered theme keywords. If yes: set `alreadyCovered: true`. If contentPlan is null (file not found), all topics get `alreadyCovered: false`.

Critical: de-duplication applies a -30% PENALTY, not exclusion. Topics still appear in output.

**Step 7 — lib/scorer.js:**

Apply scoring modifiers in order:

1. Start with topic.baseScore
2. Package match bonus: if packageMatch is non-empty and maps to a "Ready to sell" package, multiply by 1.15
3. Already-covered penalty: if alreadyCovered is true, multiply by 0.7
4. Format adjustments: twitter + short quotes = *1.05, carousel + multi-step process = *1.05
5. Store as topic.adjustedScore

Filter by --focus (match against subreddits and evidenceQuotes, case-insensitive).
Sort by adjustedScore descending.
Slice to --count.

**Step 8 — lib/formatter.js:**

Default mode (clean): For each topic, output:

```
### [Title]
**Hook:** [contentAngle or first evidence quote]
**Package:** [packageMatch or "New opportunity"]
```

Verbose mode (--verbose): Add after each topic:

```
Score: [baseScore] → [adjustedScore] | Package bonus: [yes/no] | De-dup penalty: [yes/no, reason]
Subreddits: [list] | Evidence posts: [count]
```

Output as Markdown to stdout.
