Create `research/scripts/competitor-audit.py` — a reusable CLI that audits YouTube competitor channels and produces a structured markdown report.

**Per CONTEXT.md locked decisions:**

- Audit YouTube only (Twitter skipped)
- Cover three competitor types: (1) direct small biz automation agencies, (2) automation educators/influencers, (3) no-code/low-code builders
- 5-8 competitors total — mix of direct and educators
- Output includes both gap analysis (what's NOT covered) and engagement data (what performs)
- Categorize competitor content by topic (lead gen, scheduling, invoicing, etc.) to spot saturated vs underserved topics
- Note production quality levels for benchmarking
- Flag competitors targeting the same industries (HVAC, plumbing, photo/video)

**Per RESEARCH.md (Claude's Discretion — competitor discovery):**
Hybrid approach: manually seed 3-4 known anchor search queries, then use YouTube API `search.list` to discover additional channels. Cap at 8 total.

**CLI design (argparse):**

```
python research/scripts/competitor-audit.py [options]

Options:
  --api-key TEXT         YouTube API key (overrides .env)
  --max-channels INT     Max competitor channels to analyze (default: 8)
  --max-videos INT       Max videos to fetch per channel (default: 25)
  --output-dir PATH      Output directory (default: research/output)
  --data-dir PATH        Raw data directory (default: research/data)
  --from-cache           Load from cached youtube-raw.json instead of hitting API
```

**Implementation steps:**

1. **Channel discovery function:**
   - Search YouTube API with queries: "small business automation consultant", "AI automation small business", "business process automation", "automation agency"
   - Collect unique channel IDs from search results
   - Get channel metadata (subscriber count, video count, description) via `channels.list`
   - Cap at `--max-channels` (default 8)

2. **Video collection function:**
   - For each channel, fetch top videos sorted by viewCount via `search.list`
   - Batch fetch video stats (views, likes, comments) via `videos.list` using comma-separated IDs (up to 50 per call — saves quota)
   - Store all raw data in `research/data/youtube-raw.json`

3. **Analysis function:**
   - Categorize each video by topic using keyword matching against categories: lead generation, scheduling/booking, invoicing/payments, customer follow-up, review management, intake/onboarding, general automation, no-code tools, AI/ChatGPT
   - Calculate per-channel metrics: avg views, avg likes, total videos, posting frequency
   - Calculate per-topic metrics: number of videos, avg engagement, top performers
   - Identify gaps: topics with few videos or low competition
   - Flag channels targeting HVAC, plumbing, photo/video, contractor verticals

4. **Report generation function:**
   Write `research/output/competitor-audit.md` with sections:
   - Header with generation date and script command
   - Channel Overview table: channel name, subscriber count, total videos, avg views, posting frequency, primary topics, industries targeted
   - Top Performing Content: videos with highest engagement across all channels
   - Topic Breakdown: table of topics with video count, avg views, avg engagement rate, saturation level (high/medium/low)
   - Gap Analysis: topics that are underserved, content angles competitors are NOT covering, buyer-focused opportunities (most competitors target builders, not business owners)
   - Production Quality Notes: typical video length, editing style, thumbnail patterns, CTA usage
   - Recommendations: specific content angles for @SameerAutomates based on gaps found

5. **Cache support:**
   - First run: save all API responses to `research/data/youtube-raw.json`
   - `--from-cache` flag: load from JSON cache instead of hitting API (per RESEARCH.md pitfall #4 — prevents quota exhaustion during development)

**Quota awareness (per RESEARCH.md):**

- `search.list` = 100 units/call
- `videos.list` = 1 unit/call (batch up to 50 IDs)
- `channels.list` = 1 unit/call
- Budget: ~1,700 units for full 8-channel audit. Well within 10,000/day free limit.
- Print quota usage estimate at end of run.

**Error handling:**

- If YOUTUBE_API_KEY is not set: print clear error with instructions to set it in .env or pass --api-key
- If quota exceeded (403 quotaExceeded): print error suggesting --from-cache flag for development
- If a channel search returns no results: warn and continue with remaining channels

**After building the script:**

- If the user has provided a YOUTUBE_API_KEY in research/.env, run the script to generate the report
- If not, note in the summary that the script is ready but needs the API key to produce output
